# Contributing to KK Innovation Advisor

Thanks for your interest in improving this skill!

## How to Contribute

### Report Issues

Found a bug or have a suggestion? Open an issue with:
- What you were trying to do
- What happened
- What you expected to happen

### Improve Documentation

- Fix typos or unclear explanations
- Add new examples
- Improve the references

### Add New Frameworks

KK's work is vast. If you find other frameworks or principles that fit:
- Add them to the relevant reference file
- Include examples
- Link to the original source

### Improve the Skill

- Better prompts for KK Office Hours
- Better diagnostics for Nine Laws
- New use cases or examples

## Process

1. Fork the repo
2. Create a branch (`git checkout -b improve/your-idea`)
3. Make your changes
4. Commit with a clear message
5. Push to your fork
6. Open a pull request

## Style Guide

- Keep it concise (KK would approve)
- Use examples, not abstractions
- Link to sources when possible
- Test your changes in QClaw/OpenClaw

## Questions?

Open an issue and ask. We're here to help.

---

**Remember**: This skill is about helping people think like KK. If your contribution helps with that, it's welcome.
